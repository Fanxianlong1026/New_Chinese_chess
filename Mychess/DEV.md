# 玄甲战棋 · 开发文档

> 网页版「新象棋·战棋」的上手与维护手册。配套规则原文见 `chess_rule.md`，前端美化指南见 `SKILL.md`。
> 本文覆盖：技术栈、目录、**已实现 / 待实现功能清单**、**常见数值在哪改怎么改**、关键机制、与规则原文的取舍。

---

## 1. 一句话概述

纯前端 ES 模块实现的双人同屏战棋：16×12 棋盘、HP/ATK/DEF 属性、能量系统、微型/小/奥义三级技能、堆叠压制、特殊地形。逻辑与渲染全在浏览器 JS，`serve.py` 只起一个本地 HTTP 服务。无后端、无网络依赖（除字体走 Google Fonts CDN）。

---

## 2. 技术栈与启动

- 前端：原生 ES Module + Canvas 2D，无框架、无构建步骤。
- 服务：Python 标准库 `http.server.ThreadingHTTPServer`（多线程，避免 keep-alive 长连接卡住资源加载）。
- 字体：Google Fonts（Ma Shan Zheng / Noto Serif SC / Noto Sans SC / ZCOOL XiaoWei）。

启动：

```powershell
D:\Anaconda\envs\pytorch_nightly\python.exe D:\code\MyAgent\Mychess\serve.py
# 默认 8780，端口占用自动顺延；自动开浏览器
```

验证：

```powershell
# JS 语法检查
node --check src\data.js; node --check src\engine.js; node --check src\render.js; node --check src\main.js
# 引擎回归测试（当前 58 项）
node src\_smoke.mjs
# 服务可编译
D:\Anaconda\envs\pytorch_nightly\python.exe -m py_compile serve.py
```

> ⚠️ 用浏览器自带 keep-alive 客户端（如 PowerShell `Invoke-WebRequest`）测试时，必须用本项目的 `ThreadingHTTPServer`；旧的单线程 `TCPServer` 会在第一个请求后卡住后续 `.js/.css`。

## 3. 目录结构

```text
Mychess/
├── serve.py          # 本地 HTTP 服务（多线程 / 防缓存 / 自动找空闲端口 / 自动开浏览器）
├── index.html        # 页面骨架：顶栏、棋盘容器、右侧面板、规则弹窗、结算弹窗、领域改写选择条
├── style.css         # 水墨+霓虹视觉（CSS 变量在 :root，朱红 vs 玄紫双方）
├── chess_rule.md     # 规则原文（设计源）
├── SKILL.md          # 前端美化指南
├── DEV.md            # 本文件
└── src/
    ├── data.js       # 纯数据：常量、棋子属性、初始布局、地形定义、技能表
    ├── engine.js     # 规则引擎（无 DOM）：走法、战斗、堆叠、能量、回合、地形、全部技能、胜负
    ├── render.js     # Canvas 渲染：棋盘、地形、棋子、HP/能量、高亮、预警标记
    ├── main.js       # 交互：选子、移动/攻击、技能释放、面板/弹窗、事件绑定
    └── _smoke.mjs    # 引擎回归测试（node 直接跑，断言核心规则与每个技能）
```

数据流：`main.js` 持有 `Game`(engine) 与 `Renderer`(render)，所有规则判定走 engine，UI 只读 `game` 状态并把点击翻译成 `game.act()` / `game.useSkill()`。

## 4. 已实现功能

### 4.1 核心规则
- **16×12 棋盘**，红下黑上，坐标 A–P / 1–12。
- **象棋走法**：车直、马日（蹩腿）、象田（塞田，无过河限制——战棋取舍）、炮架吃子、兵过河转向、将/士九宫。
- **战斗**：`伤害 = max(1, ATK − DEF)`；真实伤害无视护甲。击杀占位；未击杀触发**堆叠压制**。
- **堆叠压制**：每格至多 2 子（上层压制者 / 下层被压制者）；被压制者只能移动离开、不能用技能；外部攻击下层时压制者提供 50% 掩护减伤；回合结束时己方将帅被压制判负。
- **能量**：移动 +10、造成伤害 +10、受伤 +5（每回合每子至多 3 次）、能量洪流翻倍；上限 100。
- **回合流程**：一方选一个子执行「移动/攻击」或「一个技能」后换手；回合末仅结算**行动方**棋子的回血与洪流/弹坑伤害（详见 §6.2）。
- **重伤**：HP < 30% 上限时，线型棋子移动力减半、所有棋子 ATK+2。
- **胜负**：击杀将帅 / 回合末对方将帅被压制 / 认输。

### 4.2 地形（全部生效）
- **时空裂隙**（四角）：停留到自己下个回合可瞬移到任意空裂隙，瞬移即结束回合（落地眩晕）。
- **能量洪流**（中央 2×2）：能量翻倍；本方回合末受 2 真实伤害。
- **军营**（己方后两行）：回合末额外 +1 回血（共 +3）。
- **泥沼**（象·领域改写）：敌方不可进入/穿过；拥有者正常通行。
- **疾风**（象·领域改写）：己方帅/士九宫内、兵向前各 +1 步。
- **弹坑**（兵自爆/末日降临）：本方回合末对其上棋子造成 2 真实伤害。

### 4.3 技能（全部 21 个已实现）
每子 3 个（微型/小/奥义），数据见 `data.js` 的 `SKILLS`，实现为 `engine.js` 的 `_skill_<id>()`。
- 帅：急令(微·instant)、天命(小)、末日裁决(奥)
- 士：坚守(微)、灵魂链接(小)、献祭守护(奥)
- 象：伪装(微·instant)、相位转移(小)、领域改写(奥)
- 马：轻骑(微·instant)、八步赶蝉(小)、天翔龙闪(奥)
- 车：震慑(微)、毁灭冲锋(小)、末日战车(奥)
- 炮：校准(微·instant)、超电磁炮(小)、轨道轰炸(奥)
- 兵：奋进(微)、死士意志(小)、末日降临(奥)

> `instant` 类（轻骑/校准/急令/伪装）为预备增益，施放后**不结束回合**，本子可继续移动/攻击；其余技能即为该回合行动。

### 4.4 UI
顶栏（回合方/回合数/双方将帅 HP、规则/认输/新对局）、Canvas 棋盘（地形着色、合法落点/攻击/瞬移高亮、轨道轰炸预警十字、棋子 HP 条+能量环+状态点）、右侧面板（棋子详情、技能列表带消耗/冷却/全局次数、战报）、规则弹窗、胜负弹窗、领域改写「泥沼/疾风」选择条、Toast 提示。

## 6. 常见数值怎么改

> 绝大多数「平衡向」数值集中在 `src/data.js`，改完刷新页面即可（服务已禁缓存）。

### 6.1 全局常量 —— `src/data.js`（约 66–74 行）

| 常量 | 默认 | 含义 |
|---|---|---|
| `COLS` / `ROWS` | 16 / 14 | 棋盘列数/行数（第 0/13 行为祭坛行；改了要同步初始布局与地形坐标） |
| `ENERGY_CAP` | 100 | 能量上限（奥义门槛） |
| `E_MOVE` / `E_ATTACK` / `E_HURT` | 10 / 10 / 5 | 移动/造成伤害/受伤的获能 |
| `E_HURT_MAX` | 3 | 每回合每子靠受伤获能的次数上限 |
| `REGEN` / `CAMP_REGEN` | 2 / 1 | 每轮回血 / 军营额外回血 |
| `FLOW_DAMAGE` | 2 | 能量洪流真实伤害 |
| `HEAVY_WOUND_PCT` | 0.30 | 重伤触发阈值（HP 占比） |

> 想让「更容易/更难击杀」：调 `REGEN`（回血）或下方棋子 `hp`/`atk`/`def`。`REGEN=0` 则完全无自然回血。

### 6.2 棋子基础属性 —— `src/data.js` 的 `PIECE_TYPES`

```js
K: { name:'帅', nameB:'将', hp:30, atk:5,  def:0, move:'king'   },
// hp 初始/上限生命，atk 攻击，def 护甲，move 走法类型（勿随意改 move）
```
改某子强度直接改其 `hp/atk/def`。`move` 决定走法生成函数（king/advisor/elephant/horse/chariot/cannon/pawn），一般别动。

### 6.3 初始布局 —— `src/data.js` 的 `RED_SETUP`

只列红方，黑方由引擎按 `row → ROWS-1-row` 自动镜像。增删/挪动棋子改这里：

```js
{ type:'R', col:0, row:1 }, ...
```
坐标 `col 0..15`、`row 0..13`（row 0/13 = 双方祭坛行；红方底线从 row 1 开始）。

### 6.4 地形位置 —— `src/data.js`

- 时空裂隙：`RIFTS`
- 能量洪流：`ENERGY_FLOW`
- 祭坛：`ALTARS`
- 九宫范围：`PALACE`（`{cols:[6,8], rows:[1,3]}` 等）
- 河界：`RIVER`（`redCrossRow` / `blackCrossRow`，影响兵过河）
- 军营判定：`isCamp(owner, row)`

### 6.5 技能数值 —— `src/data.js` 的 `SKILLS`

每条技能：

```js
{ id:'c_railgun', name:'超电磁炮', tier:'small', cost:50, cd:2, target:'enemy', impl:true, desc:'...' }
```
- `cost`：能量消耗。`cd`：冷却回合（0 = 无）。`global`：每方全局次数（奥义常用）。
- `tier`：`micro|small|ult`（仅影响标签与门槛展示）。`target`：`self|none|cell|enemy|region`（决定是否进入选目标模式）。
- **技能内部数值**（如超电磁炮的 12 伤、天命回 10 血、轨道轰炸 20 伤）写在 `engine.js` 对应的 `_skill_<id>()` 里，改伤害/范围去那找。

### 6.6 伤害/重伤/掩护公式 —— `src/engine.js`
- 普通攻击：`_normalAttack()`（`max(1, effAtk − effDef)`，含激昂 ×1.5、校准返还、堆叠/击杀位移）。
- 攻防修正：`effAtk()`（重伤 +2、轻骑 −2）、`effDef()`（基础 + 临时护甲）。
- 重伤判定：`heavyWounded()`。
- 击退：`_knockback()`（撞墙/撞子额外 5 伤、形成堆叠）。
- 灵魂链接转移比例（40%）、献祭复活半血(6)、弹坑伤害(2)：分别在 `_dealDamage()`、`_reviveePawns()`、`_endTurn()`。

### 6.7 前端配色/视觉
- 主题色与字体：`style.css` 的 `:root` CSS 变量（`--red`/`--black`/`--jade`/`--gold`/`--rift` 等）。
- 棋盘/棋子绘制（格子尺寸 `CELL`、边距 `MARGIN`、各色值 `COLORS`）：`src/render.js` 顶部常量。

## 7. 关键机制与扩展点

### 7.1 回合模型（重要）
一「回合（turn）」= **一方动一个子的一次行动**，双方交替；`game.turn` 每行动一次 +1。`game.current` 为当前行动方。这区别于传统「一整轮」，所有「每回合」语义都按此模型实现。

### 7.2 回血时序（曾修过的坑）
回血/洪流/弹坑只在 `_endTurn()` 里对**行动方自己的棋子**结算一次——保证每子每一整轮只回一次血，且**刚被攻击的敌方不会在攻击方回合立刻回血**。不要改回「对所有棋子每手都结算」，否则会出现「杀不死人」。

### 7.3 加一个新技能
1. `data.js` 的 `SKILLS[<棋子类型>]` 加一条（`id/name/tier/cost/cd/[global]/target/impl:true/desc`）。
2. `engine.js` 加 `_skill_<id>(p, target)`，返回 `{ok, info, [instant:true]}`；扣能量/冷却/全局次数与结束回合由 `useSkill()` 统一处理。
3. 若 `target` 是 `cell/enemy/region`，在 `skillTargets()` 加一个分支返回可选目标格列表。
4. 在 `_smoke.mjs` 补断言，跑 `node src/_smoke.mjs`。

### 7.4 加一个新地形
- 位置常量进 `data.js`；着色进 `render.js` 的 `_terrain()`；效果在 `engine.js`（移动相关进 `_evalDest`/`_genLine`/`legalMoves`，回合结算进 `_endTurn`）。带归属的地形用 `this.terrain[key] = {kind, owner}`，读取用 `terrainAt()`(种类) / `terrainOwnerAt()`(归属)。

## 8. 测试

`src/_smoke.mjs` 是纯 Node 跑的引擎回归测试（不依赖浏览器），覆盖：初始布局、各类走法与阻挡、堆叠/击杀、能量、回血时序、军营、全部 21 个技能、泥沼/疾风/时空裂隙。每次改引擎后至少跑：

```powershell
node src\_smoke.mjs            # 期望：58 passed, 0 failed
```

UI/服务改动后，再用浏览器实际打开 `serve.py` 跑一局确认。
